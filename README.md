## 项目功能
- 医生管理：管理员可增删改查医生信息与排班。
- 预约挂号：用户可查看医生排班并预约挂号，支持选择日期与时间段。
- 订单管理：管理员可查看所有预约订单，用户可查看个人预约记录与状态。
- 消息通知：基于 RabbitMQ 实现订单创建与取消的异步通知处理。
- 数据库支持：使用 MySQL 存储医生、用户与订单数据，支持事务与并发控制。
- 本地开发环境：使用 Docker Compose 快速搭建后端服务与数据库环境。
- 前后端分离：后端提供 RESTful API，前端使用微信小程序进行交互。
- 日志与监控：记录关键操作日志，支持 RabbitMQ 管理插件监控消息队列状态。
- 安全与鉴权：管理员接口需登录鉴权，用户操作需验证身份。
- 可扩展性：设计考虑未来功能扩展与第三方系统对接需求。
## 目录结构
- `app/`：微信小程序前端代码。
- `backend/`：Node.js 后端服务代码。
    - `routes/`：Express 路由定义。
    - `controllers/`：请求处理逻辑。
    - `services/`：业务逻辑与数据库交互。
    - `mq/`：消息队列发布与订阅实现。
    - `config/`：配置文件与环境变量。
- `sql/`：数据库初始化脚本。
- `doc/`：项目相关文档与开发指南。
