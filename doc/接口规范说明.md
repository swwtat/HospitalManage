# 接口规范
## 通用请求体
详见 /app/utils/request.js 中的 request 函数。
所有请求均返回如下格式的 JSON：

```json
{
  "success": true,
  "data": { ... },        // 成功时返回的数据
  "error": {              // 失败时返回的错误信息
    "code": "ERROR_CODE",
    "message": "Error message details"
  }
}
```

- 特殊情况：HTTP 状态码非 200/201 时，均视为请求失败，返回格式同上，`success` 字段为 `false`。
## 鉴权
- 管理员/用户接口需在请求头中携带有效的管理员身份令牌（如 JWT），格式
    Authorization: Bearer <token>
- 鉴权失败时，返回 401 状态码及错误信息。
## 错误处理
- 所有错误均返回统一格式，包含 `code` 和 `message` 字段。
- 常见错误码：
  - `INVALID_PARAMS`：请求参数无效
  - `AUTH_FAILED`：鉴权失败
  - `NOT_FOUND`：资源未找到
  - `INTERNAL_ERROR`：服务器内部错误
- 前端应根据错误码进行相应处理，如提示用户、重试等。
## 日期与时间格式
- 所有日期时间均采用 ISO 8601 格式（UTC），如 `2024-01-31T12:34:56Z`。
- 前端在展示时可根据用户时区进行转换。
## 分页
- 列表接口支持分页查询，使用 `page` 和 `pageSize` 参数。
